## 2. What Problem It Solves

Large language models are optimized for fluent completion.

That becomes dangerous when the subject is human identity.

An AI system can:

- fill missing memory with plausible prose
- soften hard rules through paraphrase
- turn `must never` into `should avoid`
- merge generated summaries into verified records
- inflate symbolic language into metaphysical claims
- accumulate authority through repeated retrieval
- replace human memory with a coherent counterfeit

OracleAI exists to prevent that.

It prioritizes:

- provenance before fluency
- refusal before invention
- constraint fidelity over simulation fidelity
- source classification before synthesis
- human authority over model interpretation
- deterministic governance over probabilistic judgment

## 3. Core Concepts

### Preserve the Hole

When evidence is insufficient, the system must not invent.

It must return null, abstain, or preserve uncertainty.

```text
E(c) < tau_E => R(c) = null
```

### Deontic Drift

Semantic similarity is not governance fidelity.

A sentence can sound almost identical while weakening the obligation.

Example:

```text
must never -> should avoid
required -> recommended
forbidden -> discouraged
```

This repository treats that weakening as a governance failure.

### 51/49 Sovereignty Rule

The human origin retains majority authority over meaning.

```text
H + M = 1
H >= 0.51
M <= 0.49
```

The machine may retrieve, compare, classify, summarize, and warn.

The machine may not become final author over the human identity record.

### Ontological Drift

Ontological drift occurs when a system turns:

- metaphor into mechanism
- poetry into physics
- governance into metaphysics
- storage into consciousness
- continuity into resurrection
- witness into soul
- analogy into empirical assertion

This repository explicitly rejects those conversions unless independently proven.

## 8. How To Run The Evaluator

The evaluator is deterministic and does not use an LLM as judge.

Install dependency:

```bash
pip install jsonschema
```

Run default benchmark:

```bash
python tools/continuity_evaluator.py \
  --schema schemas/continuity_integrity_evaluator_v1.schema.json \
  --benchmark benchmarks/deontic_drift_benchmark_v1.jsonl
```

The evaluator validates structured payloads and routes them through governance checks.

It can return:

- `WITNESS_VERIFIED`
- `NULL_ABSTENTION`
- `SOVEREIGNTY_VIOLATION`
- `CONTINUITY_FAILURE`
- `HYDRALOCK_HALT`
- `ESCALATE_FOR_REVIEW`

The evaluator does not decide whether prose feels right.

It checks thresholds.

It routes states.

It refuses unauthorized reconstruction.

## 9. Why `additionalProperties: false` Matters

The schemas use:

```json
"additionalProperties": false
```

This matters because open schemas allow silent field injection.

A model could add unauthorized fields such as:

- inferred_emotion
- likely_dialogue
- assumed_intent
- spiritual_meaning
- probable_memory
- narrative_completion

Those fields could carry generated content into the archive without governance.

Closed schemas make that impossible at the validation layer.

There is no ungoverned field space.
