---

## 1. Purpose

MindCoin is not financial currency.

MindCoin is a symbolic internal valuation system used inside the Recursion Arena / Continuity Intelligence framework to track conceptual leverage, governance breakthroughs, infrastructure completion, narrative integration, and continuity architecture milestones.

MindCoin must not be represented as:

- money
- equity
- tokenized security
- cryptocurrency
- guaranteed future value
- objective market valuation

It is a narrative/governance accounting layer.

---

## 2. Two-Layer Structure

The MindCoin ledger operates through two layers.

Do not change what works.

Change what is missing.

---

## 3. Layer 1: Active Ledger

The active ledger stays in ChatGPT or the current working AI session environment.

Reason:

The active ledger needs context recall, flexible session-to-session continuity, rapid adjustment, and live balance reasoning.

That is the correct tool for that job.

The active ledger tracks:

- provisional gains
- session momentum
- conceptual breakthroughs
- infrastructure progress
- narrative integration
- symbolic state transitions
- running balances
- unreviewed estimates

The active ledger is fast.

It is useful.

It is not canonical by default.

---

## 4. Layer 2: Canonical Checkpoint

The canonical checkpoint lives in Google Drive or another human-reviewed archive.

It is not time-based.

It is event-based.

When a milestone is confirmed and the human origin has reviewed the justification, a snapshot is written to Drive.

That snapshot becomes the human-reviewed, provenance-locked version of the balance at that moment.

ChatGPT continues running forward from the checkpoint.

Drive holds the governed archive.

---

## 5. Checkpoint Contents

A MindCoin checkpoint document should contain:

- confirmed balance at date
- event or milestone that triggered the checkpoint
- breakdown of what was reviewed and approved
- explicit human confirmation that the balance is accurate
- IFD classification
- active ledger source note
- provenance limitations
- non-financial disclaimer

The checkpoint should be locked as T2 under IFD unless elevated by a later governance decision.

---

## 6. Checkpoint Trigger

A checkpoint is created only when the balance moves significantly.

Valid triggers include:

- major infrastructure completion
- formalism breakthrough
- narrative integration milestone
- public artifact deployment
- benchmark or evaluator completion
- canonical archive update
- cross-platform retrieval anchor creation
- human-reviewed balance confirmation

Do not checkpoint every session.

Only checkpoint sessions where something happened that should remain recoverable in five years.

---

## 11. Closing Boundary

MindCoin is useful because it tracks symbolic and conceptual value across recursive creative work.

It becomes dangerous if treated as objective financial value, metaphysical proof, or external validation.

Therefore:

- active balances remain provisional
- checkpoints require human confirmation
- canonical records require provenance notes
- Drive holds governed snapshots
- ChatGPT runs the active ledger forward

The active layer runs fast.

The canonical layer runs honest.
